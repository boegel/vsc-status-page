Current incidents

Tier-2 scratch issues

Start: 2023-11-29 10:00:00
End: Unknown
Updated: 2023-12-27 09:25:25
Level: low

Problems with tier-2 scratch performance Users may experience low responsiveness of the scratch storage on Genius and wICE. The issue with the infiniband network is identified. To mitigate the probleit will probably include frequent reboots of a number of nodes including the login nodes.

  • update 01/12/2023A preventive reservation has been set on the whole cluster tarting Monday morning. This is just in case we need a larger scale reboot of the file system, which is not yet sure. For now jobs that can't finish before Monday will be held in the queue untill the reservation is released. Jobs that are running will keep on running.
  • update 01/12/2023 The reservation has been moved to Wednesday. No special errors are reported
  • update 04/12/2023 The source of the problem is not yet clear, but there were no hardware errors reported. There will be no full reboot of the system needed, so the reservation will be released. We will keep on monitoring the system, performance might still be more variable. In order to check for undetected broken network links, the network will be further examined, while the system is under load.
  • update 05/12/2023 The system appears to be stable since 10:30 yesterday, with three network switches disabled. The network throughput is currently one quarter of the normal capacity. From tomorrow morning, December 6th, we will reenable switches gradually, to see when the problems reappear so we can locate possible broken parts.
  • update 18/12/2023 The system has been to be stable, only one network switches is disabled. The network throughput is normal.

Planned maintenance

Tier-2 wICE maintenance

Start: 2024-01-03 08:00:00
End: 2024-01-04 08:00:00
Updated: 2023-12-27 09:25:25
Level: high

Tier-2 wICE needs a full maintenance window. The ethernet switches will be updated and rebooted. This will temporarly break connections to the netapp (VSC_HOME, VSC_DATA en apps). In the same time window a slurm update will be applied. Jobs that can't end before January 3th 08:00 will remain in the queue. The system will be released as soon as possible after the update and appropriate tests have been run. During this maintenance, Genius will remain operational.
